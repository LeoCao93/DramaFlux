# DramaFlux 开放平台浏览器布局优化设计

## 目标

在保留现有品牌 Logo、SVG 图标、代码高亮、轨道装饰、边框与蓝紫光效的前提下，将首页、接口文档和定价页调整为更适合真实浏览器窗口的统一布局系统。

设计不再以 1680×945 单一设计稿的逐像素填满为目标。主要验收尺寸为 1366×768、1440×900 和 1680×945，并兼顾平板与手机。

## 统一布局系统

- 三页共享 `1280–1440px` 自适应内容容器、统一 Header 高度、圆角、字号层级和 8px 间距基准。
- 使用 `clamp()` 控制容器边距、标题字号、区块间距和卡片尺寸，避免在普通窗口中出现过密内容或大块空白。
- 首页与定价页由内容自然撑开，允许约 0.3–0.8 屏的正常滚动，不使用固定页面高度强制塞入首屏。
- Hero 插画和代码窗按容器宽度缩放，不通过绝对定位扩大文档宽度。
- 宽屏维持首页三列能力卡和定价五列套餐卡；中等窗口逐步缩小并换列，小屏回到单列。
- 页脚紧随页面内容，不使用固定底部或额外最小高度制造留白。

## 页面设计

### 首页

- 保留单行主标题、代码窗口、能力卡、接口卡和定价引导卡。
- Hero 使用自适应双栏；空间不足时先缩小间距和字号，再切换为上下结构。
- 代码窗口使用可读的最小高度，长代码在窗口内部滚动，不影响整页宽度。
- 各内容区使用统一垂直节奏，避免设计稿比例在浏览器中形成上下断层。

### 接口文档

- 保留工作台模式：左侧目录和右侧在线调试固定，中间文档内容独立滚动。
- 工作台宽度与营销页视觉边界对齐；在宽屏使用三列，在中等窗口压缩列宽，在窄屏转为纵向文档流。
- 在线调试响应结果使用带行号的 JSON 语法高亮，并保留状态、耗时和响应大小。
- 预定义参数与自定义参数共用参数行样式，但维护独立身份。

### 定价页

- 保留 Hero、权益、VIP 标题条、五张套餐卡、购买时长和底部开通栏。
- 周付、月付、季付、半年付、年付分别使用不同 SVG 图标；周付和月付不得复用同一图形。
- 套餐卡在不同桌面宽度下保持相同比例和信息层级，不强制固定为设计稿像素宽度。
- 定价页 Header 不显示“立即接入”，避免与页面内购买主按钮重复。

## 导航与品牌

- 首页和接口文档页 Header 显示“立即接入”，目标统一为 `/pricing`。
- 定价页隐藏 Header“立即接入”，使用页面内“立即开通 VIP”作为唯一主要购买动作。
- 使用 DramaFlux 六边形品牌图标生成 SVG favicon。
- 页面标题分别为：
  - 首页：`DramaFlux 开放平台`
  - 接口文档：`接口文档 - DramaFlux`
  - 定价页：`定价购买 - DramaFlux`

## 在线调试参数行为

- 每个参数行包含参数名、类型、值和删除按钮。
- 删除预定义或自定义参数时，仅从当前调试请求中移除，不修改接口目录元数据。
- 删除必填预定义参数后发送请求，继续使用现有请求校验并显示明确的缺失参数提示。
- “添加参数”创建新的空白自定义参数行。
- 自定义参数支持填写参数名、选择 `string` 或 `boolean` 类型及填写值。
- 自定义参数名为空时不发送；名称重复时以内置参数或前一条有效参数为准，并显示冲突提示。
- 有效自定义参数作为 query 参数加入实际请求 URL，不改变 path 参数替换规则和后端接口结构。
- 切换接口时清空当前接口的自定义参数，并恢复目标接口的预定义参数。

## 测试与验收

- 验证三个页面在 1366×768、1440×900、1680×945 下无横向溢出，内容占比与垂直节奏一致。
- 验证 820px 和 390px 宽度下导航、卡片和文档工作台正确换列。
- 验证首页与定价页没有由固定高度产生的大块空白。
- 验证文档页左右栏稳定、中间栏独立滚动，窄屏转为整页滚动。
- 验证添加、编辑、删除自定义参数会正确改变实际请求 URL。
- 验证删除必填参数会触发校验，切换接口会重置参数状态。
- 验证调试响应 JSON 的行号和字符串、键、数字、布尔值、空值高亮。
- 验证 Header CTA 的显示规则、目标路由、三页标题和 favicon。
- 运行完整前端测试、TypeScript 检查和生产构建，并使用实际 API 完成一次在线调试。

## 范围边界

- 不新增后端接口，不修改 API 响应结构。
- 不实现真实支付、账户体系或 API Key 发放。
- 不将自定义参数写入接口目录或持久化到浏览器存储。
- 不取消现有视觉细节，只调整其尺寸、布局和浏览器适配策略。
